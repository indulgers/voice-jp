# voice-jp 安装与使用指南

> 给完全不熟悉编程的朋友的安装手册。预计总耗时 **15–25 分钟**（含模型自动下载）。

## 你需要准备的东西

- 一台 Mac（Apple Silicon，M1/M2/M3/M4 均可）
- 大约 4GB 可用硬盘空间（模型 3GB + 缓存）
- 微信账号（可以扫码登录）

---

## 步骤 1：装 WeFlow（微信桥）

voice-jp 自己不会连微信。它需要 WeFlow 帮忙读取微信里的语音消息。

1. 打开 https://github.com/hicccc77/WeFlow/releases
2. 下载最新版的 `WeFlow-x.x.x-arm64.dmg`
3. 双击 dmg，把 **WeFlow.app** 拖进 **Applications** 文件夹
4. 打开 Applications，**右键点击 WeFlow → 「打开」**（双击会被系统拦截，必须右键）
5. 弹出"无法验证开发者"的对话框，点 **「打开」**
6. 在 WeFlow 里**扫码登录微信**（用手机微信扫一下）
7. 登录成功后，打开 WeFlow 的 **设置**：
   - 勾选 **「HTTP API」** （开启接口服务）
   - 勾选 **「Message Push」** （开启消息推送）
   - 复制旁边显示的 **access_token**（一长串字母数字，例如 `0123456789abcdef0123456789abcdef`）—— 等下要粘贴

保持 WeFlow 开着，**不要关**。

---

## 步骤 2：装 voice-jp（一条命令搞定）

1. 打开「**终端**」app（聚焦搜索 ⌘+空格 → 输入 `terminal` → 回车）
2. 把下面这**一整行**复制粘贴进去，按回车：

   ```
   /bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/indulgers/voice-jp/main/install.sh)"
   ```

3. 等大约 1 分钟（看网速）。脚本会：下 dmg → 装到 Applications → 启动 → 自动开浏览器
4. 看到 `✓ 完成！` 字样后，浏览器应该会自动弹出 `http://localhost:7788`

如果浏览器没自动弹，自己打开这个地址。

> 💡 为什么要在终端里跑这条命令，不能双击 dmg？
> 因为 voice-jp 是免费的个人项目，没付苹果 $99/年的签名费。从 Chrome 下载的 dmg 装出来会被 macOS Sequoia 一律判 **"已损坏"**，连「右键 → 打开」都救不了。
> 这一行的作用就是**绕开浏览器**，用 `curl` 直接从 GitHub 拿 dmg —— `curl` 下载的文件不会被 macOS 标记。安全的，只是无法走 Chrome 那条路。
>
> 以后想升级新版本，**同样跑这一行**就行。

## 步骤 3：跟随浏览器向导（4 步）

浏览器里会出现一个引导页：

### 第 1 步 — 「启动 WeFlow」
应该自动显示 ✓（绿色）。如果显示红色"还连不上 WeFlow"，说明 WeFlow 没在跑，回去步骤 1 检查。

### 第 2 步 — 「粘贴 access_token」
粘贴你在 WeFlow 设置里复制的那串 token，点「保存并验证」。

### 第 3 步 — 「下载语音识别模型」
点「开始下载」。会下载约 3GB 的语音识别模型到本地。**5–15 分钟**，看你网速。下载条会实时更新进度，不用守着。

### 第 4 步 — 「选择要监听的联系人」
- **语言** 选「日语」（如果你要转中文就选「中文」）
- 在搜索框输入你日本朋友的微信名 / 备注 / 微信号，从列表里点选 —— 可以选多个人
- 点「保存并完成」

完事了！主界面会出现。

## 使用

只要 WeFlow 和 voice-jp 都开着，**白名单内联系人**发的每条语音消息都会自动出现在 voice-jp 的浏览器界面里，几秒后显示转写文字。可以播放原音、复制文字。

我推荐：把 voice-jp 的浏览器窗口拖窄、放在屏幕角落、按 Cmd+Shift+T 之类的快捷键固定（你的浏览器支持的话）。

## 调整设置

想加/减联系人，或改语言：在 voice-jp 的浏览器界面里点 **「⚙ 设置」** 展开，重新选完点保存。

## 关闭和重启

- **关 voice-jp**：在 Dock 上右键 voice-jp → 退出
- **关 WeFlow**：同理
- 下次想用，分别打开就行。**不会自动开机启动**（这是有意的，避免后台一直跑）

## 数据保存在哪

聊天转写结果保存在你 Mac 上的：`~/Library/Application Support/voice-jp/`

包含：
- `voice-jp.db` — 转写文本数据库
- `cache/` — 下载的语音文件
- `models/ggml-large-v3.bin` — 3GB 模型文件

要彻底清除就把这个文件夹删了。

## 常出问题

| 现象 | 检查 |
|---|---|
| 浏览器顶部红字"连不上 WeFlow" | WeFlow 没开 / 退出了 / 没登录 → 重启 WeFlow，重新登录 |
| 朋友发了语音但没出现 | 1) 微信 PC 端要登录并接收到这条消息 2) 这个联系人不在白名单里 → 设置里加上 |
| voice-jp 打开报"已损坏，无法打开" | 你应该是手动从 GitHub 网页下了 dmg 装的。**重跑步骤 2 那一行 curl 命令**就能修好（重装 + 自动去 quarantine） |
| 模型下载失败 | 重新点「开始下载」会自动续传 |
| 想转其他语言（不是日语） | 设置里改「语言」下拉，选日语 / 中文 / 自动检测 |
| 想升级新版本 | **重跑步骤 2 那一行 curl 命令**就行 |

遇到任何其他问题，截图发我。

## 附录：手动 dmg 安装（不推荐，仅给好奇的人）

如果你坚持要从 GitHub Releases 网页下 dmg 自己装：

1. 下 https://github.com/indulgers/voice-jp/releases/latest/download/voice-jp.dmg
2. 双击 dmg，把 voice-jp.app 拖进 Applications
3. 必须在终端跑：`xattr -cr /Applications/voice-jp.app`（否则装完报"已损坏"）
4. 双击启动

`install.sh` 做的事跟上面一样，只是自动化 + 没第 3 步陷阱。除非你要研究脚本，否则用 curl 一行更省事。
